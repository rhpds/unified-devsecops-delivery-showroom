== Welcome

This hands-on workshop demonstrates how to deliver applications through a *secure, standardized, and automated path to production* using the *Red Hat Advanced Developer Suite (RHADS)*.

You will experience how platform engineering enables developers to move fast, while ensuring that every change is trusted, verified, and compliant.

---

== About Red Hat Advanced Developer Suite

Red Hat Advanced Developer Suite (RHADS) enables organizations to standardize how applications are built, secured, and delivered across hybrid and multi-cloud environments.

It provides a unified platform that balances developer productivity with software supply chain security, embedding governance and trust directly into the delivery process.

---

=== Core Benefits

* **Standardized Path to Production**  
Developers follow a consistent, approved workflow to build and deliver applications, regardless of environment.

* **End-to-End Trust and Provenance**  
Every artifact is signed, verified, and traceable, ensuring integrity across the entire software supply chain.

* **Built-in Security and Compliance**  
Policies are enforced automatically throughout the pipeline, reducing risk without introducing manual gates.

* **Full Visibility and Risk Awareness**  
SBOM analysis and vulnerability insights provide clear visibility into software composition and risk.

=== Key Components Covered

image::rhads-architecture.png[rhads-architecture,align="center",link=self,window=blank]

[cols="1,3"]
|===
| Component | Description

| *Red Hat Developer Hub (RHDH)*
| Provides a central developer portal with Software Templates that offer a self-service, standardized way to create and deliver applications.

| *Red Hat Trusted Artifact Signer (RHTAS)*
| Enables keyless signing and verification of artifacts, ensuring integrity and trusted provenance.

| *Red Hat Trusted Profile Analyzer (RHTPA)*
| Provides visibility into software composition, vulnerabilities, and compliance through SBOM analysis.

| *Red Hat Dev Spaces*
| Delivers cloud-based development environments with preconfigured tooling, enabling consistent developer workflows.
|===

---

=== Supporting Capabilities

[cols="1,3"]
|===
| Component | Description

| *Red Hat Build of Keycloak (RHBK)*
| Provides identity and access management, enabling OIDC-based authentication and keyless signing workflows.

| *Red Hat Advanced Cluster Security (RHACS)*
| Enforces security policies and performs vulnerability scanning across the software lifecycle.

| *Red Hat Quay*
| Stores container images and associated artifacts, including signatures, attestations, and SBOMs.
|===

== Workshop Structure

This workshop is structured around key challenges organizations face when securing software delivery:

* *Module 1: Establish Software Composition Trust with SBOMs*  
Understand what is in your software and identify risk early.

* *Module 2: Sign and Verify Artifacts*  
Ensure every artifact is trusted, signed, and tamper-evident.

* *Module 3: Developer Workflow Without Friction*  
Enable developers to move fast using Software Templates and self-service workflows.

* *Module 4: Enforce Policy and Promote Safely*  
Ensure only trusted and compliant software is promoted across environments.
