= Module 2: Sign and Verify All Artifacts With *RHTAS*

== Introduction 

In the previous module, you established visibility into what is inside your software by analyzing SBOMs. The next challenge is ensuring that what gets built and delivered has not been altered along the way.
In modern software delivery, artifacts move across multiple systems, build pipelines, registries, and environments. Without a way to verify their origin and integrity, organizations are exposed to risks such as tampering, unauthorized changes, or compromised supply chains.

This is where *Red Hat Trusted Artifact Signer (RHTAS)* comes in. *RHTAS* provides a platform capability for signing and verifying software artifacts, allowing organizations to confirm that what they are deploying was produced by a trusted process and has not been modified.

In this module, you will see how trusted signing is embedded directly into the platform. As artifacts are built, they are automatically signed, creating a verifiable record of their origin and integrity as they move through the delivery lifecycle.

== Step 2: Validate artifact signing

* In this step, you will configure and run a simple CI pipeline that builds and signs an artifact, confirming that signing is working as expected.
* *Tekton* is the pipeline engine used in this workshop. A *Task* represents a unit of work, such as building and pushing a container image, while *Tekton Chains* extends the pipeline to automatically sign and verify artifacts.
* Navigate to the Showroom Terminal and click the green play button to run the script that configures *Tekton Chains* and the pipeline task:
+
[source, role="send-to-wetty"]
----
./lab-assets/setup_image_sign.sh
----
+
* Once setup is successful, execute the following script to run the pipeline task:
+
[source, role="execute"]
----
./lab-assets/test_image_sign.sh
----
+
* This script triggers athe *Tekton Task* that builds and pushes a container image. Because *Tekton Chains* was configured in the previous step, it will automatically:
. Detect the successful task completion.
. Sign the resulting image using *Cosign*.
. Store the signed image and its signature in the container registry (*Quay*).
* *Quay* is the container registry used in this workshop to store images and their associated signatures.
* This validates that signing is correctly integrated into the pipeline and that artifacts can be traced and verified.
* In the OpenShift console, navigate to: Home → Projects  
* Select the project: `tssc-app-ci`
* From the left-hand menu, select: Pipelines → TaskRuns
* You should see a TaskRun associated with the signing test.
+
image::taskrun_view.png[TaskRun View,align="center",link=self,window=blank]
+
* Select the TaskRun and review its execution.
* Once the task completes successfully, the resulting artifact is signed automatically by the platform.
